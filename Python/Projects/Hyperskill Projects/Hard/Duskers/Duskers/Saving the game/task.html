Stage 5: Description

Theory

Before you can start implementing the stage, it might be a good idea to go over some theory.

In order to check if a file exists in a given directory, we can use the os.path module. The syntax is as follows: 

os.path.isfile(path)

It returns a boolean value. We can use a relative path here to refer to a file in the same directory as our main script.

For example, os.path.isfile("save_file.txt") looks for a file named save_file.txt in the same folder as your script. If such a file, exists it returns True. Make sure you implement it this way for the tests.

Description

In this stage, we will take care of something very important in games: saving the game to a file and loading it. The program will also record the date and time of the save. How you format the save file is completely up to you, you may do something as simple as storing it in a plain text file, or use a popular format such a JSON. The only requirement is that the string save_file is in the file name of the file(s) you use to store the game state. The tests are only concerned about if the saved data made its way back into the program after loading a game, i.e. the file structure is not actually checked.

Objectives

  1. We will use 3 save slots. These contents of these slots should be displayed when selecting the Load option. If there is no data in a save slot, it should display empty. The save slots are numbered (from 1 to 3), and the player enters the number of the slot they want to load or save to. Make sure that the player is not able to load an empty slot! Each slot, if populated, should display the player's name, their current Titanium balance, and Robot count. It should also contain the current time and date of saving in the following format: YYYY-MM-DD hh:mm. For example: 2020-09-24 20:05.

    To summarize, this is the structure of the slot as seen in-game: 

    <player-name> Titanium: <balance> Robots: <number-of-robots> Last save: <date-and-time>
    Where text in <angled-brackets> is a placeholder.

    Example: 

    Hyperskill Titanium: 50 Robots: 3 Last save: 2022-01-15 19:29
    Remember: how you structure the actual save files is up to you. Only the displayed data inside the load menu and the actual loaded game state will be tested.

  2. Make sure the game state is actually loaded in the game
  3. Display a small dialog box if saving/loading was successful.
  4. Make sure you also add the saving functionality to the save and exit option. Similarly here, three slots should be available to choose from, and the player should select the slot to save to by entering its number.
  5. You should now replace Play with New game and Load game options in the menu.
