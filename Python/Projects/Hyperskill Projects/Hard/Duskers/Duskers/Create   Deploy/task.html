Description

We promised robots: it's time for robots! In this stage, you should create your own robots using websites such as fsymbols.com to help with the design. For the tests, make the robots exactly 7 lines tall. After finishing your design, hit the "COPY" button and it's ready to be pasted into your code! You will also add a menu to the game screen.

Objectives
  1. Your robots are represented by their ASCII/Unicode images, which should be 7 lines tall. In this stage, you should have three robots.
  2. Place the robots into the Hub you made earlier. Make sure each robot's string representation is rectangular and that they are identical to one another, including any whitespaces.
  3. Underneath the robots, insert 4 options just like in the examples below:
    • [Ex]plore
    • [Save]
    • [Up]grade
    • [M]enu
    where the text in brackets represents what the player needs to type in to execute that command. We will be implementing these options in later stages.
  4. For now, just add the [M]enu to our game screen. The menu should have the following options:
    • [Back] to game
    • Return to [Main] Menu
    • [Save] and exit
    • [Exit] game
    Make a nice little box around your menu like the one in the example. Again, same thing applies, the word in brackets should be typed in by the user if they want to select that option. Remember, for not-yet implemented options display a placeholder message and terminate the program.
  5. The menu options should do what they say, except for [Save] and exit: just insert a placeholder message there and end the program. We will deal with saving later.
Examples

The greater-than symbol followed by a space (> ) represents the user input. Note that it's not part of the input.
