Description

Tamagotchi is an electronic pet simulator. It was created in 1996 in Japan and quickly gained popularity all over the world. This small digital pet has several well-being parameters: level of hunger, happiness, health, and others. The point of the game is taking care of your pet: for example, feeding it to satisfy its hunger and playing games to make it happier.

In this project, you will write a special kind of Tamagotchi called Robogotchi where the pet is not an animal but a robot. Even though it's not a living creature, it still requires care and attention: only instead of feeding it you charge it, instead of putting it to bed you switch it to the sleeping mode, and instead of meds it needs oil.

Before creating the robot itself, let's think about how we will interact with it. As responsible pet owners, we need to make sure our robot doesn't get lonely or bored.

In this stage, we will implement a simple game called Numbers. Here are the rules: first, a random number is generated in the interval from 0 to 1,000,000. Then, the robot player and the human player each provide a number. The player whose number is closer to the generated number wins. Inputting exit game closes the game.

Prompt the user with a message What is your number?. Then, show the result in the following format:

The robot entered the number N.
The goal number is K.

Depending on the result, print one of the following messages: You won!, The robot won!, or It's a draw!.

Note that the user's number has to fall in the given range. If the user inputs a negative number, the program should print The number can't be negative!. If the user provides a number larger than 1,000,000, print Invalid input! The number can't be bigger than 1000000. Also, the user has to input a numeric value, so strings shouldn't be allowed. If the user inputs a string, the program should print the following message: A string is not a valid input!.

Finally, let's treat our users to some statistics. When the user exits the game, the stats should be printed in the following format: 

You won: {number of your wins},
Robot won: {number of the robot's wins},
Draws: {number of the draws}.

Delete the stats when starting a new game

Objectives

In this stage, your program should:

  1. Take the user's input, check its validity, and ask again for a valid input if needed.
  2. If the user provides an invalid option, the program has to display an appropriate message. All three invalid options — negative numbers, text strings, and numbers over 1,000,000 — have to be handled differently.
  3. Output the result of the game.
  4. Keep taking the input as long as the user wants to play.
  5. Collect the game statistics and print it when the user decides to exit the game.
