Description

Now that we have some good entertainment prepared, it is time to create the robot itself. In this stage, we will write an interface that will help us interact with the robopet. Your robot should get a name and have an opportunity to play, recharge, and cool off. The user should be able to get all the information about their robopet and quit the game.

So, let's talk about the robot's features. All of them except for the name will change during the robot's lifetime; however, in the beginning, they have predefined values. Here are the robot's basic features:

  • name; the program prompts the user for the robot's name with the message How will you call your robot?;
  • level of battery is set to 100;
  • level of overheat is set to 0;
  • level of skills is set to 0;
  • level of boredom is set to 0.

There are a few ways that the user can interact with their robopet and improve its well-being parameters:

  • play allows the user to play one of the two possible games. Playing decreases the level of boredom by 20 and increases the level of overheat by 10 after the user enters 'game over'. To help the user keep track of the changes, print the following messages for them:

    {name of the robot}'s level of boredom was {previous value}. Now it is {current value}.
    {name of the robot}'s level of overheat was {previous value}. Now it is {current value}.

If the level of boredom equals 0, print this message after all the messages described above:

{name of the robot} is in a great mood!

Do not forget to display the game stats after the game is played

  • recharge gives the robot a chance to boost its energy. If the battery level is 100, print {name of the robot} is charged!. Otherwise, the option Recharge increases the level of battery by 10, decreases the level of overheat by 5, and increases the level of boredom by 5. Display a suitable message for the user:

    {name of the robot}'s level of overheat was {previous value}. Now it is {current value}.
    {name of the robot}'s level of the battery was {previous value}. Now it is {current value}.
    {name of the robot}'s level of boredom was {previous value}. Now it is {current value}.
    {name of the robot} is recharged!
  • sleeping mode gives the robot a well-deserved break and allows it to cool off. If the level of overheat is 0, print the message {name of the robot} is cool!. If the level of the overheat is equal to 0 after sleeping, print the message about changes of the values and then print the previous message. Otherwise, entering the sleeping mode decreases the level of overheat by 20. Print an appropriate message for the user:

    {name of the robot} cooled off!
    {name of the robot}'s level of overheat was {previous value}. Now it is {current value}..
  • info prints all the information about the robopet and its vitals. Here is the expected format:
{name of the robot}'s stats are:
battery is {value},
overheat is {value},
skill level is {value},
boredom is {value}.
  • Exit lets the user quit the game. Print the message Game over. and exit the game.

The values of all the attributes fall in the interval from 0 to 100. These are not merely numbers: the vitals tell us something about the pet's well-being, so some of them influence the behavior of the robot. Below we described the possible effect that overheat can have on the robot. Note that in the next stage several more effects will be added. They are listed in the order of preference, which means that if several of them happen at the same time, the first action to actually happen in the game should be the one higher in this list.

  • If the level of overheat reaches 100, the robot explodes, and the program prints the following message: The level of overheat reached 100, {the name of the robot} has blown up! Game over. Try again?. Then, the game ends.

Note that the possibility to exit the game and see the stats should always be available during the game, even if the robot's battery is at 0 or its boredom level is 100.

The interface works as follows: when the game starts, the user is prompted to enter the robot's name with the message How will you call your robot?. Then, the program should show the user the options:

Available interactions with {name of the robot}:
exit - Exit
info - Check the vitals
recharge - Recharge
sleep - Sleep mode
play - Play

If the user enters an invalid option, the program should notify the user by printing Invalid input, try again!. The list of possible interactions should then be shown again. Keep displaying this list after each interaction with the robot.

The robot's values cannot be negative

Objectives

In this stage, your program should:

  1. Provide the following interactions: playing, recharging, sleeping;
  2. Print the information about the robot: its level of battery, boredom, overheating, and skill;
  3. Provide an option to quit the game;
  4. Take the user's input;
  5. If the input is correct, the chosen interaction should be performed. Otherwise, the program should inform the user and ask them for a valid input;
  6. React when certain parameters reach critical levels.
