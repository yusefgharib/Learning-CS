Description

Let's see what our game is actually about. You have robots under your command (3 to start with), and your mission is to explore various locations where you will find titanium that is used as a currency and encounter occasional enemies.

In this stage, we will add more elements to our game: some other menu items and a simple graphic hub. The Hub is basically a game screen where the player will see the most important information.

If you cannot wait, you may also start adding simple text animation elements. However, we won't be checking this until stage 4, so make sure this functionality is deactivated for the tests.

Objectives
  1. Add two new options to our menu: high scores and help. For now, they won't be very functional: we will work on them later. The new display order should be as follows: Play, High scores, Help, Exit.
  2. If the user chooses [High] scores, show them a message that there are no scores to display.
  3. Back should bring the user to the previous screen. Display this option on the high scores screen.
  4. Whenever a feature isn't available yet, display some placeholder text to inform the user that the feature is coming soon, and end the program.
  5. Add a possibility to return to the main [Menu] when the player is asked if they are ready to begin the game.
  6. After starting the game, display a HUB like the one in the first example below. For now, it is just a placeholder.

You can also add some text animation using the time.sleep module if you'd like. Feel free to play around with this, just make sure to disable animation for the test.

Note that command words should be in brackets so that the player knows what to type. Please use the same command words that we're using in the descriptions and examples. This applies to all stages of the project.

Examples

The greater-than symbol followed by a space (> ) represents the user input. Note that it's not part of the input.
